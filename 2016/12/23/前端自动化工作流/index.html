1.1 Node环境

1.1.1 node.js

  • Server side javascript runtime–服务端的js运行时
    • 目前js运行在浏览器内核中
    • php既是一门脚本语言也是一个运行环境
  • node.js可理解为js的运行环境，可以在node中运行JS代码
    • JS由ECMAScript、DOM、BOM三部分组成,node中只能运行ECMAScript,无法使用BOM和DOM
  • node将浏览器JS引擎（chrome的V8引擎）搬到了服务器端，增加了一些提供文件、网络之类操作的API
  • node.js内置http服务器（PHP需要Apache才能运行）,十几M的文件并发量超乎想象

1.1.2 目前有两个分支

  • Node.js 0.12.7 官方版本（完美主义者）
  • IO.js 是社区的产物，不是官方的(两个编写逻辑和代码一模一样，IO.js有很多
    新特性，迭代非常快，社区推进很快)
  • 15年两者合并，发布了node第一个正式版4.0(后来又分开了)
    • node 5.x==IO.js
    • node 4.0==node

1.1.3 nvm（node version manager）

  • 因为node版本比较多，开发人员可能依赖很多版本，使用node版本管理工具nvm进行切换node版本
  • Node.js是一个轻内核（本身没有什么功能）的东东，所有的功能都要功能包提供
    node官方提供了一些最基础的包

1.1.4 npm（ Node Package Manager）

  • Node应用程序依赖包的管理工具，进行安装卸载更新之类的操作
  • 常见的包管理工具都有循环依赖的功能，只需记住要什么东西
  • 安装一个包，默认安装最新稳定版本

1.2.3 环境变量

  • 环境变量就是操作系统提供的系统级别用于存储变量的地方
  • 环境变量分为
    • 系统变量和用户变量
    • 系统变量指所用当前系统用户共享的变量
    • 自己的电脑一般只有一个用户（建议将自己配置的环境变量放在用户变量中，用户变量比较干净）
  • 环境变量的变量名不区分大小写（变量间运行相互引用）
  • 特殊值：
    • PATH变量（不区分大小写）
    • PATH 相当于一个路径的引用（添加到PATH变量中的路径,可以在任何
      目录下搜索）
  • 命令行（可以用来执行当前目录下的文件）
    • 命令（cd :change directory）

  • 环境变量配置

    • （1）nvm环境变量配置，路径一直到可执行文件nvm.exe和node.exe
    • （2）npm环境变量配置，路径一直到Windows命令脚本文件npm.cmd

  • 将环境变量加入到PATH路径

  • 查看命令行所使用的的环境变量

1.5 node工具集

1.5.1 Bower（web应用程序依赖项管理工具） 类似于npm

  • 官网
  • Web sites are made of lots of things — frameworks, libraries, assets, and utilities. Bower manages all these things for you.
  • Bower就是用来管理项目中所有的依赖，主要用于Web页面开发时使用的包管理，比如jquery，bootstrap（而npm主要用于引用node的包）
  • 方便便捷的方式管理包（不推荐，除了安装官方的包还可以安装其他的包）
    • 修改npm全局路径，就是在用户目录下添加.npmrc文件（npm config list查看是否配置成功）
    • bower的使用需要有git环境

1.5.2 Gulp（当下最流行的自动化工具）

  • 不再是简简单单的使用HTML+CSS+JavaScript这些简单的技术构建网页应用程序了，我们要提高效率，就必须减少重复的工作，使用less之类预处理的CSS coffeescript
    • 提供开发阶段的便利，开发阶段更快捷
    • 现在的开发行业优质的开发人员是不应该将精力放在这些重复性质的工作上
  • Gulp就是一种可以自动化完成我们开发过程中大量的重复工作
    • 预处理语言的编译
    • js css html 压缩混淆
    • 图片体积优化
  • 除gulp之外还有一些类似的自动化工具，比如grunt
    • what how why
  • 什么是自动化构建工具？
    • 自动完成一系列重复的操作：less → css、coffeescript → js、css压缩、js混淆、html压缩、img尺寸优化……
  • 链接：
    • 官网
    • 中文网

  • gulp的包既包含工具，也包含一些编程的API

  • gulp就是用来机械化的完成重复性质的工作，gulp的机制就是将重复工作抽象成一个个的任务

  • 执行相关任务所需的插件

  • 常用插件

  • 编译 Less：gulp-less
  • 创建本地服务器：gulp-connect
  • 合并文件：gulp-concat
  • 最小化 js 文件：gulp-uglify
  • 重命名文件：gulp-rename
  • 最小化 css 文件：gulp-minify-css
  • 压缩html文件 gulp-minify-html
  • 最小化图像：gulp-imagemin

1.5.3 grunt（和gulp类似）

  • 百度结合gulp和grunt做了FIS3（重复的轮子太多，抓住轮子与轮子之间的共性）
  • grunt的速度比gulp慢，并且grunt在运行过程中会产生一些临时文件。gulp所有的操作都是在内存中完成的，gulp更加好用、更普遍，用法类似

1.5.4 git（一个分布式源代码管理工具）

  • 源代码
    • 在一个项目中，凡是由开发人员编写的都算是源代码
  • 源代码有必要管理起来？
    • 让源代码可以被追溯，主要记录每次变更了什么，谁主导这次变化，人为的维护比较麻烦
  • 源代码管理工具便于大家协同开发
  • git
    • GIT是Linux之父当年为了维护管理Linux的源代码写的一个工具
    • Git 之前 很多使用 svn vss tfs hs ……
  • git的环境变量

1.5.4 GitHub（一个著名的git服务提供商）

  • 官网
    • 提出社交化编程
  • git服务器
    • 用于同步本地的代码仓库
    • git是一个工具，GitHub是一个网站
    • “免费”服务，免费服务必须是开源的
  • gist
    • github上可直接贴小的代码段，可以创建私有的gist，还可以复制对应的链接到网页或博客，以模块的形式载入代码段
